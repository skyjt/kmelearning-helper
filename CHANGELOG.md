# 更新日志（Changelog）

本文件记录 KME 学习助手（kmelearning-helper / KME Learning Navigator）各版本的主要变化。
格式参考 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，版本号遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## [2.5.0] - 2026-06-07

### 变更（Changed）
- 行为开关收进浮窗的**翻转设置面**：点右上角 **⚙**，卡片会 3D 翻转到背面（设置），点「**完成**」再翻回正面。正面只保留「开始 / 停止自动学习」「立即检查」和进度显示，更清爽。
- 设置面包含四项开关：自动播放、未完成自动补学、总时长达标再返回、跳过做题页；改动**即时生效**。
- 浮窗的「总进度」只在**课程目录页**或**自动学习进行中**显示；在其它页面或未学习时不再显示上次留下的残留进度。

### 移除（Removed）
- 去掉「**锁定 1x 真实学习**」开关。视频本就按平台规则固定以 1x 播放，该开关没有实际意义；1x 现为自动学习的固定核心行为，不再可关闭。

## [2.4.1] - 2026-06-06

### 修复（Fixed）
- 修复课程目录里**同名课程被合并**、导致总数少算的问题。去重逻辑原先只按文字判重，会把标题完全相同的两门课（例如两门「征信合规管理」）当成一门，进度条因此显示 `5/6` 而非 `6/6`。现改为按「文字 + 屏幕位置」区分：同名但位于不同位置的课程会分别保留与计数；嵌套/重叠的同一行仍会被正确合并。

### 已知限制（Known issue）
- 课程完成状态目前仍按**标题**记录，因此两门完全同名的课程在「自动学习」中还不能分别追踪——第二门可能被当成已完成而跳过。计划在后续版本改为「标题 + 出现次序」识别。

## [2.4.0] - 2026-06-06

### 变更（Changed）
- 浮窗不再在所有 `pc.kmelearning.com` 页面默认弹出，改为**默认最小化成右下角的图标**。
- 仅当进入学习地图的**课程目录页**（路径 `…/home/training/study`）时，浮窗才会**默认自动展开**；课程播放页、答题页、首页等其它页面保持最小化，不再遮挡视频或正文。
- 平台是单页应用，自动学习在「目录 → 课程 → 返回目录」之间跳转时，浮窗会**随页面自动展开 / 最小化**；在同一页面手动展开或最小化的选择会被保留，直到再次切换页面。

### 备注
- 课程正文页归属 `…/home/course/…`，已与目录页区分，因此默认保持最小化。
- 升级后请在 `chrome://extensions` 重新加载扩展。

## [2.3.0]

### 优化
- 性能优化：对 DOM 扫描与 `innerText` 读取按需缓存（epoch / TTL），大幅减少每次轮询和面板刷新的重复开销。
- 面板局部刷新：启停状态只更新按钮与状态圆点，不再整块重建浮窗。
- 界面焕新：开关改为紧凑拨动样式、悬浮图标改用 128px 高清图标，并补充深色模式与「减少动态效果」适配。

## [2.2.0]

### 新增
- **课程总时长校验**：返回课程列表前，检查学习记录里的「学习总时长」是否达到课程要求学时，不够则按 1x 继续补学 / 重播。
- 新增「总时长达标再返回」开关。

## [2.1.0]

### 变更
- 更新安装说明与扩展元数据（核心逻辑与 2.0.0 一致）。

## [2.0.0]

### 首个公开版本
- 在学习目录页注入「学习助手」浮窗，实时显示当前目录的总进度。
- 按目录顺序自动进入未完成课程并播放，锁定 1x 真实学习。
- 视频播完但平台未确认完成时，自动重播补学。
- 自动跳过考试 / 测验 / 问卷 / 作业等需人工完成的页面（不代答、不提交）。
- 文档 / 材料页保持活跃，等待平台完成标记。
